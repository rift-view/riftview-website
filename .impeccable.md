# RiftView Marketing Site — Design Context

## Product
RiftView: a desktop app (Electron/React) that scans a cloud account and renders it as a live connected graph. Solves fragmentation in the provider console — "what else is connected to this?" answered in 10 seconds instead of 4 tabs. Core features: blast radius, live cross-service graph, top risks, drift detection (vs Terraform), guided remediation. Marketing site is cloud-provider agnostic in language.

## Users
DevOps engineers, SREs, platform engineers. Context: on-call at 2am, production is breaking, they have minutes to find the cross-service dependency that's cascading. They hate marketing fluff. They trust technical density.

## Job-to-be-done
Website must convince a skeptical SRE, in under 90 seconds, that this tool will save them during their next incident — enough to download a desktop app.

## Brand personality (3 words)
Forensic. Kinetic. Unhedged.

- Forensic = incident-investigator energy, not dashboard-vendor energy
- Kinetic = the product is about LIVE state, the site should feel alive, not static
- Unhedged = confident claims backed by specifics, no corporate caveats

## Aesthetic direction
The product is named after a geological *rift* — a fault line where plates separate and you can see what's underneath. The site leans into that metaphor: a faint seismic/fault-line motif, a live node graph as the hero (not a generic dashboard screenshot), and editorial pacing where dense technical sections alternate with short, bold statements.

**Theme:** Dark. Non-negotiable for this audience and use context. SREs live in dark terminals; an incident tool in a bright white marketing wrapper would feel tonally wrong.

**Palette:** Near-black base (deep ink, tinted slightly warm/brown — NOT the AI-default cyan-on-pure-black), bone/cream for primary text, one signature accent pulled from warning-amber (the color of a live alert), and a secondary technical accent in oxidized copper/bronze for subtle UI rhythm. Explicitly avoiding: neon cyan, purple-to-blue gradients, glowing drop shadows, any AI-dev-tool cliché.

**Typography:** Display face with presence and a little strangeness (condensed grotesque or a geometric with sharp terminals). Body in a refined humanist sans. Mono for data blocks and CLI snippets. Pair chosen to feel like a technical essay in a good print journal, not a SaaS dashboard.

**Motion:** Restrained. One orchestrated hero reveal where the node graph draws itself. Hover states on nodes that ripple blast-radius through connections. No bounce, no elastic, no scattered micro-interactions.

## Anti-references (what this must NOT look like)
- Vercel / Linear / Stripe gradient dashboards (too generic)
- Glowing cyan-on-pure-black dev tools (the AI cliché)
- Enterprise B2B SaaS templates (hero image of diverse team smiling at laptop, etc.)
- Corporate feature-grid with identical cards

## Design principles
1. **Live over static.** The hero demonstrates the product with motion, not a screenshot. The site itself *is* the first demo.
2. **Specifics over adjectives.** "10 seconds" beats "blazing fast." "3 highest-severity chain-of-failure risks" beats "insightful alerts."
3. **Editorial rhythm.** Dense technical blocks (mono, data, CLI) alternate with bold editorial statements. The pace should feel like reading a sharp essay.
4. **The rift is the motif.** A single vertical/diagonal seam runs through the design language — alignments break across it, content on either side relates. Used sparingly, never decoratively.
5. **Earn every animation.** Motion only where it clarifies: graph drawing, blast radius propagation, drift diff. No motion for motion's sake.
